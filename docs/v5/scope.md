# Scope V5

## Vision

La V5 prolonge la V2, la V3 et la V4 deja en place sans rouvrir le tunnel
d'achat ni le post-achat.

La priorite devient la clarification du modele produit pour mieux refléter
deux cas simples :

- produit `simple`
- produit `variable`

L'objectif est de rendre le catalogue et l'admin plus lisibles, sans basculer
vers un systeme produit complexe.

## Point de depart

La base actuelle permet deja :

- de gerer un catalogue administrable
- de vendre des declinaisons porteuses de SKU, prix, stock et disponibilite
- d'utiliser une boutique achetable de bout en bout
- d'exploiter les commandes et le post-achat

La V5 ajoute la souplesse qui manque encore au modele produit :

- type produit explicite
- lecture simple ou variable plus coherente
- admin produit plus facile a comprendre

## Inclus

- type produit explicite `simple` / `variable`
- compatibilite avec les produits variables deja presents
- lecture catalogue coherente selon le type
- simplification de l'admin produit selon le type
- migration douce du modele existant

## Exclu

- systeme configurable complexe
- attributs generiques type WooCommerce
- bundles, kits ou packs
- options produit avancees
- matrices multi-dimension complexes
- refonte large du tunnel d'achat

## Priorites

1. clarifier le type reel de chaque produit
2. conserver les produits variables existants sans rupture
3. permettre un produit simple achetable sans detour inutile
4. rendre l'admin produit plus lisible selon le type
5. garder une lecture publique coherente et previsible

## Criteres de fin V5

La V5 est consideree utile quand :

- un produit peut etre explicitement `simple` ou `variable`
- les produits variables existants continuent de fonctionner sans rupture
- la lecture publique respecte clairement le type du produit
- l'admin produit n'expose que les informations utiles selon ce type
- le modele reste petit, lisible et sans logique produit complexe
