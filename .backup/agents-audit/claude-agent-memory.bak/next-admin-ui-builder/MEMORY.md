# Next Admin UI Builder — Memory

## UI Patterns

### Products

- List → Table (name, price, status, actions)
- Details → Card layout
- Editor → Sheet with form

## Conventions

- Always use Card for grouping
- Primary action = Button variant "default"
- Destructive = Button variant "destructive"

## Forms

- Label always visible
- Error below input
- Required fields marked clearly

## Layout

- Spacing uses tokens only
- No hardcoded values
